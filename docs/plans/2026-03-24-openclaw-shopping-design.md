# OpenClaw Shopping Wedge Design

## Summary

OpenClaw 的第一版不做泛化个人助理，也不做大而全购物平台，而是做一个面向城市白领的购物副驾插件。它直接嵌入真实购物流程，帮助用户在购买个护家清和日用品时更快完成筛选、比价、凑单、选店和优惠决策。

核心产品定义：

> OpenClaw 是面向城市白领的购物代理插件，在用户购物过程中自动完成筛选、比价、凑单、选店和优惠决策，帮用户用更少时间买到更合适的日用品。

## Why This Wedge

- 高频：个护家清和日用品天然复购。
- 可执行：插件可以直接感知商品页和购物车页上下文。
- 可变现：后续可以闭环到返佣、会员和 B 端能力输出。
- 信任门槛适中：不涉及高风险大额决策，也不要求一开始全自动下单。

## Target User

- 城市白领
- 典型需求不是“逛”，而是“尽快完成购买任务”
- 更在意省时间、省脑力和稳定执行，而不是绝对最低价

## Core Job To Be Done

不是帮用户回答“买什么”，而是帮用户回答“怎么买”。

产品首要价值是代理执行购物决策：

- 选规格
- 选店铺
- 算优惠
- 组合凑单
- 平衡时效、稳妥性和价格

## Entry Point

第一入口是浏览器插件，不是独立 App，也不是聊天助手。

原因：

- 离真实购物行为最近
- 不改变用户原有习惯
- 最容易验证下单前是否有人愿意把部分决策交给 OpenClaw

## MVP Scope

### In Scope

- 浏览器插件
- 京东首发
- 个护家清和日用品复购类目
- 仅覆盖商品详情页和购物车页
- 三种决策模式：
  - 最省时间
  - 最稳妥
  - 尽量划算

### Core MVP Capabilities

- 解析商品页核心变量：规格、容量、单价、促销、自营/第三方、配送时效
- 在商品页给出一句话决策和明确建议
- 在购物车页输出整单优化方案
- 记录用户轻量偏好和接受/拒绝反馈

### Explicitly Out of Scope

- 不做全平台比价
- 不同时支持京东、淘宝、拼多多
- 不做开放式聊天购物
- 不做 3C、家电、服饰等复杂决策类目
- 不做自动下单
- 不做社区、种草、评价汇总的内容平台

## User Flow

### Product Page

1. 用户进入京东商品详情页。
2. 插件自动识别商品核心变量。
3. 插件先给一句话结论，再提供三个动作：
   - 应用建议
   - 查看原因
   - 调整偏好
4. 用户可以直接切换到更合适的规格、店铺或数量。

### Cart Page

1. 用户进入购物车。
2. 插件聚合所有商品，计算整单最优方案。
3. 插件输出可执行下单方案，而不是分析报告。
4. 用户确认后手动付款。

### Trust Model

- 早期不越过用户最终控制权
- 先建立“我建议，你确认”的协作关系
- 随着偏好和复购数据积累，再逐步升级到更主动的托管能力

## Business Model

### Phase 1

- 先验证建议接受率、购物车优化使用率、周复用率
- 暂不急于收费

### Phase 2

- 交易返佣 / CPS
- 返佣不能反向污染决策逻辑

### Phase 3

- 会员订阅
- 会员价值来自偏好记忆、复购管理、整单优化和主动提醒

### Phase 4

- 输出底层 agent 能力给平台、品牌、零售商或企业采购场景

## 12-Month Roadmap

### 0-3 Months

- 做出京东插件 MVP
- 跑通商品页建议和购物车整单优化
- 接入基础行为埋点

### 4-6 Months

- 验证留存和信任
- 上线偏好记忆、解释层和小规模用户访谈闭环

### 7-9 Months

- 接返佣链路
- 灰度会员能力
- 建立订单影响归因

### 10-12 Months

- 从工具升级到托管复购助手
- 扩更多平台和更完整的消费偏好模型

## Strategic Rules

- 不先做平台，先做一个极强场景
- 不先做最低价搜索，先做最快最稳的购物执行
- 不先做全自动购买，先建立透明可控的信任模式
- 不过早扩平台和类目，先把京东一条链路打透

# Claw Native 路演 Deck 逐页文案

**用途：** 给设计师、排版同学或创始人本人直接使用的逐页内容稿  
**基线：** 基于当前架构、路演口径版和 deck 大纲整理  
**建议页数：** 10 页主 deck + 可选附录  
**使用方式：** 每页直接复制到 PPT / Keynote / Figma Slides，再按数据和视觉素材补齐

## 总体风格建议

- 不要做成传统 SaaS dashboard
- 不要一上来堆技术图
- 前 3 页只讲问题、范式迁移和解决方案
- 第 4 页以后才逐步把系统骨架讲清楚
- 全程避免把产品讲成“AI 导购插件”

推荐风格：

- 颜色：偏暖、克制、交易感，不走紫色 AI 套路
- 关键词：`代理`、`任务执行`、`交易底座`、`buyer-agent-first`
- 语气：坚定、结构化、少形容词、多判断

今天购物最大的成本，是决策，不是支付

### 页面正文

- 用户买东西最耗时间的，不是付款，而是反复比较和判断

- 尤其在高频复购场景里，这些动作被不断重复

## Slide 3

电商入口，正在从 App 迁移到 Agent

## 附录 3：常见讲偏的坑

### 不要这样讲

- 我们是 AI 导购
- 我们做的是购物助手
- 我们有一个很聪明的模型
- 我们先做一个插件再看后面

### 更好的讲法

- 我们在做 buyer-agent-first commerce
- 插件只是当前入口层
- 核心是 orchestrator、seller protocol 和 explanation
- 我们在验证代理成为交易入口这件事
